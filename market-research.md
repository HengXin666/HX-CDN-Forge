# 市场上类似项目调研报告

## 调研结论

经过全面调研，**市面上没有与本项目完全相同的产品**。现有项目主要分为以下几类，但都存在明显差异：

---

## 一、现有项目分类

### 1. CDN 负载均衡器（服务端）

#### CDN Selector
- **GitHub**: https://github.com/cdnexperts/cdnselector
- **技术栈**: JavaScript (90.2%)
- **Star 数**: 24
- **核心功能**:
  - 多 CDN 流量分割（支持自定义权重）
  - 即时故障转移
  - 认证令牌转换
  - 基于客户端网络的路由
- **目标场景**: 流媒体视频服务（HLS、HDS）
- **部署方式**: 服务端中间件
- **与我们项目的差异**:
  - ❌ 不是前端组件，是服务端应用
  - ❌ 无延迟测速功能
  - ❌ 无可视化节点选择界面
  - ✅ 支持多 CDN 负载均衡

#### cdn-load-balancer
- **GitHub**: https://github.com/vano468/cdn-load-balancer
- **技术栈**: Ruby (85.2%)
- **Star 数**: 0（项目不活跃）
- **核心功能**: 流量分发、健康检查
- **与我们项目的差异**:
  - ❌ 服务端实现，非前端组件
  - ❌ 项目已不活跃
  - ❌ 无用户交互界面

---

### 2. 构建工具插件

#### rollup-plugin-cdn
- **GitHub**: https://github.com/WebReflection/rollup-plugin-cdn
- **技术栈**: JavaScript
- **状态**: **已废弃**
- **核心功能**:

  - 支持 unpkg.com、rawgit.com
- **与我们项目的差异**:
  - ❌ 仅用于构建时处理，非运行时选择
  - ❌ 无延迟测试功能
  - ❌ 无用户选择界面
  - ❌ 项目已废弃

---

### 3. CDN 故障切换方案（简单实现）

#### 前端 CDN 降级方案
- **来源**: SegmentFault 技术问答
- **实现方式**:
  ```javascript
  var cdnArr = [
    {css: '//unpkg.com/bootstrap@3.3.7/dist/css/bootstrap.css',
     js: '//unpkg.com/bootstrap@3.3.7/dist/js/bootstrap.min.js'}
  ];
  
  function autoCDN() {
    var cdn = cdnArr.shift();
    // 通过 onerror 实现简单的降级
  }
  ```
- **核心功能**: 当 CDN 失败时，自动切换到下一个
- **与我们项目的差异**:
  - ❌ 仅支持故障降级，无主动测速
  - ❌ 无延迟对比
  - ❌ 无用户选择界面
  - ❌ 无持久化存储
  - ❌ 实现过于简单，非完整组件

---

- **类型**: CDN 服务提供商
- **功能**: 前端公共资源库加速

#### cdnjs
- **GitHub**: https://github.com/cdnjs/cdnjs
- **类型**: CDN 内容库
- **功能**: 托管开源前端库
- **与我们项目的差异**:
  - ❌ 是资源库，非选择器工具

## 三、目标用户和使用场景

### 现有项目的目标用户
1. **CDN Selector**: 流媒体服务提供商、需要多 CDN 负载均衡的企业
2. **rollup-plugin-cdn**: 需要在开发时使用 CDN、打包时回退本地的开发者
3. **简单降级方案**: 需要基本 CDN 容错的小型项目

### 我们项目的目标用户
1. **GitHub 开源项目维护者**：需要为中国用户提供快速访问
2. **个人博客/文档站**：使用 GitHub 托管静态资源
3. **前端开发者**：需要可靠的 GitHub CDN 加速方案
4. **出海项目**：同时服务国内外用户

## 四、竞争优势

### 1. 填补市场空白
- 目前**没有专门针对前端 GitHub CDN 资源的智能选择器**
- 现有方案要么是服务端，要么是构建时，缺少运行时智能选择

### 2. 用户体验优化
- 可视化界面让用户了解各节点性能
- 自动选择最佳节点，无需用户干预
- 支持手动切换，灵活性高

### 3. 技术优势
- 完整的 TypeScript 支持
- React 组件化设计
- 开箱即用的默认配置
- 基于真实测速数据选择

### 4. 针对性强
- 专为 GitHub 资源优化
- 内置国内用户友好的 CDN 节点
- 解决 jsDelivr 在国内访问不稳定的问题

## 六、结论

**本项目具有明确的市场定位和独特价值**：
- 现有方案多为服务端或构建时方案，缺乏前端运行时智能选择
- 本项目是**首个**提供可视化界面、实时测速、自动选择的 GitHub CDN 选择器组件
- 填补了前端 GitHub CDN 资源智能管理的空白
- 具有良好的开源潜力和实际应用价值

建议以 **CDNForge** 或 **CDNRadar** 作为项目名称，突显创新性和专业性。
